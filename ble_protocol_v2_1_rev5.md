# CTM BLE 通信协议文档

**版本:** V2.1 (Rev.5)  
**适用设备:** CTM Modbus 控制器 (ESP32-C3)  
**通信方式:** BLE 5.0 (Nordic UART Service)  
**更新日期:** 2026-05-05

---

## 1. BLE 服务信息

| 项目 | 内容 |
|------|------|
| 设备广播名 | CTM=XXXXXXXXXXXX (12位蓝牙MAC HEX大写) |
| NUS Service UUID | 6E400001-B5A3-F393-E0A9-E50E24DCCA9E |
| RX 特征 UUID | 6E400002-... (Write/WriteNR, App->设备) |
| TX 特征 UUID | 6E400003-... (Notify, 设备->App) |
| MTU | 建议 247+, 最大 500 |
| 配网方式 | BLE NUS 指令 (CMD 0x0B) / SmartConfig (CMD 0x0C) |

**连接流程:**

1. 扫描广播, 过滤名称前缀 CTM=
2. 连接设备
3. 发现 NUS Service
4. 使能 TX Notify (写 CCC = 0x0001)
5. 通过 RX 发送指令帧, 通过 TX Notify 接收应答帧

设备断开后自动恢复广播, App 可重新扫描连接.

---

## 2. 帧格式

```
HEAD(0x68) + CMD(1B) + LEN(2B小端) + DATA(NB) + CS(1B) + TAIL(0x16)
```

| 字段 | 长度 | 说明 |
|------|------|------|
| HEAD | 1 | 固定 0x68 |
| CMD | 1 | 指令码, 应答帧 CMD 或 0x80 |
| LEN | 2 | 数据域长度, 小端 |
| DATA | N | 数据域 |
| CS | 1 | SUM(HEAD ~ DATA末字节) & 0xFF |
| TAIL | 1 | 固定 0x16 |

## 4. 指令详情

### 4.1 MQTT 开关 (CMD: 0x01)
请求 DATA (1B): D0: 0x00=关闭, 0x01=开启  
应答 DATA (1B): D0: 0x00=成功, 0x0F=失败

### 4.2 TG服务器 开关 (CMD: 0x02)
格式与 CMD 0x01 相同. TG服务器默认启用, 无密钥时状态码为 0x07.

### 4.3 读取 MQTT 配置 (CMD: 0x03)
请求 DATA: 无  
应答 DATA (210B): URI(80B) + Port(2B小端) + 用户名(32B) + 密码(32B) + Topic(64B)

### 4.4 写入 MQTT 配置 (CMD: 0x04) ★ Rev.3 修复

请求 DATA (210B): 布局同 CMD 0x03  
应答 DATA (1B): 0x00=成功, 0x0F=失败(长度不足或URI为空)

**Rev.3 修复:** URI空校验; port=0自动使用1883; URI端口拼接改为检查"://"后是否已有":"

### 4.5 读取全部连接状态 (CMD: 0x05) ★ Rev.5

应答 DATA (**17B**):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | WiFi 状态 (见状态码表) |
| D1 | 1 | WiFi RSSI (int8, dBm) |
| D2 | 1 | MQTT 使能: 0=禁用, 1=启用 |
| D3 | 1 | MQTT 状态 |
| D4 | 1 | TCP服务器 使能: 0=禁用, 1=启用 |
| D5 | 1 | TCP服务器 连接状态 |
| D6 | 1 | Modbus 连接状态 |
| D7-D10 | 4 | MQTT 上次上报时间 (uint32 小端, Unix秒) |
| D11 | 1 | TG密钥配置: 0=未配置, 1=已配置 |
| D12 | 1 | TCP综合配置状态: 0=未配置, 1=已配密钥(默认TG), 2=完整配置 |
| D13 | 1 | SNTP 使能: 0=关闭, 1=开启 |
| D14 | 1 | SNTP 同步状态: 0=未同步, 1=已同步, 2=同步中, 3=同步失败 |
| **D15** | **1** | **TCP 服务器模式: 0=自定义TCP, 1=TG服务器** |
| **D16** | **1** | **TCP 加密状态: 0=明文, 1=AES加密** |

**WiFi 状态码:** 0x00=未配置, 0x01=连接中, 0x02=已连接, 0x03=密码错误, 0x04=AP未找到, 0x05=认证失败, 0x06=连接超时, 0x07=SmartConfig中, 0x08=BLE配网中

**MQTT 状态码:** 0x00=已禁用, 0x01=DNS失败, 0x02=TCP连接失败, 0x03=鉴权失败, 0x04=连接中, 0x05=已连接, 0x06=已断开

**TG服务器 连接状态码:** 0x00=已禁用, 0x01=DNS失败, 0x02=连接失败, 0x03=连接中, 0x04=已连接, 0x05=已断开, 0x06=等待WiFi, 0x07=无密钥

**Modbus 状态码:** 0x00=空闲, 0x01=通信正常, 0x02=超时, 0x03=CRC错误, 0x04=异常

**SNTP 同步状态码:** 0x00=未同步(SNTP未启用或从未同步), 0x01=已同步, 0x02=同步中, 0x03=同步失败(网络不可达)

### 4.6~4.7 MQTT 上报间隔 (CMD: 0x06/0x07)
读取: 应答 4B uint32 小端  
写入: 请求 4B, 范围 10~360 秒

### 4.8 读取设备信息 (CMD: 0x08) ★ Rev.3

应答 DATA (**46B**):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0-D5 | 6 | BLE MAC 地址 |
| D6-D37 | 32 | 固件版本 (取自 main.c firmware_rev_val, 如 "V1.1.3") |
| **D38-D41** | **4** | **运行时间 (uint32 小端, 秒)** |
| **D42-D45** | **4** | **设备时间 (uint32 小端, Unix秒, 0=NTP未同步)** |

### 4.9~4.10 TCP服务器 配置 (CMD: 0x09/0x0A) ★ Rev.5

读取 84B: Host(80B) + Port(2B小端) + server_mode(1B) + encrypt_enabled(1B)

| 偏移 | 长度 | 说明 |
|------|------|------|
| 0-79 | 80 | 服务器主机名/IP |
| 80-81 | 2 | 端口号 (uint16 小端) |
| **82** | **1** | **server_mode: 0=自定义TCP, 1=TG服务器** |
| **83** | **1** | **encrypt_enabled: 0=明文, 1=AES加密 (TG模式下强制为1)** |

写入 82~84B, 持久化, 运行中自动重连.

**兼容性:** 旧版 APP 发送 82B (仅 Host+Port), 固件保持当前 server_mode 和 encrypt_enabled 不变.

**自定义 TCP 模式 (server_mode=0):** 默认 tgc-tcp-uat.capax-tech.com:9000, 不需要 app_key, 加密由 encrypt_enabled 控制.

**TG 模式 (server_mode=1):** 默认 smartdevice.towngas.com:10020, 需要 app_key (通过 SmartConfig 写入), 强制 AES 加密.

### 4.11 WiFi 配网 (CMD: 0x0B)
请求: SSID_len(1B) + SSID(N) + Pass_len(1B) + Pass(M)  
应答: 0x00=已开始, 0x0F=参数错误  
注意: BLE 配网不写入 TG 密钥.

### 4.12 触发 SmartConfig (CMD: 0x0C)
应答 0x00. SmartConfig 成功后自动写入 TG 密钥.

### 4.13 设备重启 (CMD: 0x0D)
应答后 500ms 重启.

### 4.14 恢复出厂设置 (CMD: 0x0E)
清除所有 NVS 数据并重启. **不可逆, App 应做二次确认.**

### 4.15~4.16 TG服务器 上报间隔 (CMD: 0x10/0x11)
与 MQTT 上报间隔格式相同, 范围 10~360 秒.

### 4.17 读取 Modbus 设备状态 (CMD: 0x20) ★ Rev.3

应答 DATA (**145B**):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | 设备类型数量 N (固定 9) |
| D1~ | N×16 | 每设备 16 字节 |

**每设备 16 字节结构:**

| 偏移 | 长度 | 说明 |
|------|------|------|
| +0~+7 | 8 | 设备名称 (ASCII, 0填充) |
| +8 | 1 | 从机地址 |
| +9 | 1 | 使能: 0=禁用, 1=启用 |
| +10 | 1 | 通讯状态: 0=空闲, 1=正常, 2=超时, 3=CRC, 4=异常 |
| +11 | 1 | 保留 |
| +12~+13 | 2 | 当前寄存器起始地址 (uint16 小端) |
| +14~+15 | 2 | 当前寄存器数量 (uint16 小端) |

**设备类型列表:**

| 索引 | 名称 | 从机地址 | 默认寄存器 |
|------|------|----------|-----------|
| 0 | GAS_FLOW | 0x01 | 0x0001, 13个 |
| 1 | FRIDGE | 0x02 | 0x0000, 21个 |
| 2 | CURRENTWA | 0x03 | 0x0020, 6个 |
| 3 | DRYCONTAC | 0x04 | 0x0000, 3个 |
| 4 | DISHWASHE | 0x08 | 0x1320, 42个 |
| 5 | GAS_CONC | 0x09 | 0x0016, 10个 |
| 6 | ULTRASONI | 0x0A | 0x0000, 8个 |
| 7 | STEAMER | 0x19 | 0x0000, 5个 |
| 8 | WOK | 0x1A | 0x0000, 3个 |

### 4.18 设置 Modbus 设备通讯使能 (CMD: 0x21) ★ Rev.3

请求 DATA (2B): D0=设备索引(0~8), D1=使能(0/1)  
应答 DATA (1B): 0x00=成功, 0x0F=失败  
持久化到 NVS.

### 4.19 读取 Modbus 通讯日志 (CMD: 0x22) ★ Rev.3

应答 DATA (变长, 最大 321B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | 日志条数 N (0~10) |
| D1~ | N×32 | 每条 32 字节 |

**每条日志 32 字节结构:**

| 偏移 | 长度 | 说明 |
|------|------|------|
| +0~+3 | 4 | 时间戳 (uint32 小端, Unix秒) |
| +4~+19 | 16 | 设备名称 (ASCII, 0填充) |
| +20 | 1 | 通讯结果: 0=成功, 1=超时, 2=CRC, 3=异常 |
| +21 | 1 | 从机地址 |
| **+22** | **1** | **操作类型: 0=读保持寄存器, 1=读输入寄存器, 2=读离散输入, 3=写保持寄存器** |
| +23 | 1 | 保留 |
| **+24~+25** | **2** | **寄存器起始地址 (uint16 小端)** |
| **+26~+27** | **2** | **寄存器数量 (uint16 小端)** |
| +28~+31 | 4 | 保留 |

App 显示建议:
- 操作类型用图标区分: 📖读保持 📋读输入 🔘读离散 ✏️写保持
- 寄存器信息格式: "Addr: 0x0001, Cnt: 13"
- 结果: 绿色=成功, 红色=失败

### 4.20 修改 Modbus 设备读取地址 (CMD: 0x23) ★ Rev.3

修改指定设备类型的 Modbus 寄存器读取地址, 立即生效.

请求 DATA (5B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | 设备索引 (0~8) |
| D1-D2 | 2 | 新的寄存器起始地址 (uint16 小端) |
| D3-D4 | 2 | 新的寄存器数量 (uint16 小端, 1~125) |

应答 DATA (1B): 0x00=成功, 0x02=参数无效(数量超范围), 0x0F=索引越界

修改持久化到 NVS, 重启后保持. 修改后立即重新加载 Modbus 描述符, 下一个轮询周期使用新地址.

示例 (设备0 GAS_FLOW, 改为起始地址 0x0010, 读取 8 个寄存器):
```
发送: 68 23 05 00 00 10 00 08 00 CS 16
                  |idx=0|addr=0x0010|cnt=8|
应答: 68 A3 01 00 00 CS 16
```

### 4.21 修改 Modbus 设备从机地址 (CMD: 0x24) ★ Rev.3

修改指定设备类型的 Modbus 从机地址 (slave address), 立即生效.

请求 DATA (2B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | 设备索引 (0~8) |
| D1 | 1 | 新的从机地址 (1~247) |

应答 DATA (1B): 0x00=成功, 0x02=参数无效(地址超范围), 0x0F=索引越界

修改持久化到 NVS, 重启后保持. 修改后立即重新加载 Modbus 描述符, 下一个轮询周期使用新地址.

示例 (设备1 FRIDGE, 从机地址改为 0x05):
```
发送: 68 24 02 00 01 05 CS 16
                  |idx=1|addr=5|
应答: 68 A4 01 00 00 CS 16
```

### 4.22 每日校时开关 (CMD: 0x25) ★ Rev.3

控制设备是否启用每日自动校时功能 (SNTP).

请求 DATA (1B): D0: 0x00=关闭, 0x01=开启

应答 DATA (1B): 0x00=成功, 0x0F=失败

设置持久化到 NVS. 启用后设备每天通过 SNTP 自动同步网络时间, 关闭后设备时间不再校准.

### 4.23 读取每日校时状态 (CMD: 0x26) ★ Rev.4

读取每日校时当前使能开关和同步状态.

请求 DATA: 无

应答 DATA (**2B**):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | SNTP 使能: 0=关闭, 1=开启 |
| D1 | 1 | SNTP 同步状态: 0=未同步, 1=已同步, 2=同步中, 3=同步失败 |

App 进入页面时应先发送此指令, 将 SNTP 开关 UI 初始化为设备当前状态.

### 4.24 读取当前 WiFi 配置 (CMD: 0x27) ★ Rev.4

从设备读取当前已保存的 WiFi SSID 和密码.

请求 DATA: 无

应答 DATA (**97B**):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | SSID 长度 N (0~32, 0=未配置) |
| D1-D32 | 32 | SSID (UTF-8, 0填充) |
| D33 | 1 | 密码长度 M (0~64) |
| D34-D97 | 64 | 密码 (UTF-8, 0填充) |

如果设备从未配网, SSID 长度为 0, App 显示 "未配置".

### 4.25 读取 Modbus 轮询间隔 (CMD: 0x28) ★ Rev.4

请求 DATA: 无

应答 DATA (**4B**): uint32 小端, 单位毫秒

Modbus 轮询间隔指主机对所有已使能从设备的轮询周期. 默认 2000ms.

### 4.26 写入 Modbus 轮询间隔 (CMD: 0x29) ★ Rev.4

请求 DATA (4B): uint32 小端, 单位毫秒, 范围 **500~10000**

应答 DATA (1B): 0x00=成功, 0x02=参数无效(超范围), 0x0F=失败

修改持久化到 NVS, 立即生效.

### 4.27 读取最近 MQTT 上报报文 (CMD: 0x2A) ★ Rev.4

读取设备最近缓存的 MQTT 上报 JSON 报文, 最多 5 条.

请求 DATA: 无

应答 DATA (变长, 最大 2561B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | 报文条数 N (0~5) |
| D1~ | N × 变长 | 每条报文: Len(2B小端) + JSON(UTF-8) |

**每条报文结构:**

| 偏移 | 长度 | 说明 |
|------|------|------|
| +0~+1 | 2 | JSON 长度 L (uint16 小端, 最大 512) |
| +2~+(L+1) | L | JSON 报文 (UTF-8, 不含\0) |

设备缓存最近 5 条已发送的 MQTT 报文, 新报文覆盖最旧记录. 报文按时间倒序 (最新在前).

App 显示建议: 以列表展示, 每条报文用等宽字体显示 JSON 内容, 支持长按复制.

## 8. Modbus 设备管理 (Rev.3)

### 8.1 设备状态监控
CMD 0x20 一次性获取所有设备的名称、地址、使能、通讯状态和当前寄存器配置.

### 8.2 设备通讯开关
CMD 0x21 控制各设备是否参与轮询. 持久化.

### 8.3 地址修改
CMD 0x23 运行时修改寄存器起始地址和数量. 持久化, 立即生效.

### 8.4 从机地址修改
CMD 0x24 运行时修改从机地址. 持久化, 立即生效. 从机地址范围 1~247.

### 8.5 通讯日志
CMD 0x22 查看最近通讯事件. 每条记录包含操作类型(读/写)、目标寄存器地址和数量, 便于精确定位问题.

## 9. 通用错误码

| 值 | 含义 |
|----|------|
| 0x00 | 成功 |
| 0x01 | 分区/校验错误 |
| 0x02 | 空间不足/参数无效 |
| 0x03 | 数据不完整 |
| 0x0F | 参数错误/数据长度不足/索引越界 |
| 0xFF | 未知错误/无活动OTA会话 |

## 11. 注意事项

1. **字节序:** 所有多字节整数小端, OTA 文件大小和进度为大端
2. **状态轮询:** CMD 0x05, 建议 2~3 秒间隔
3. **上报间隔:** MQTT 和 TG 各自独立, 范围 10~360 秒
4. **TG 密钥:** 必须通过 SmartConfig 写入
5. **Modbus 地址修改:** CMD 0x23 修改寄存器地址, CMD 0x24 修改从机地址, 均持久化+立即生效; reg_count 范围 1~125, slave_addr 范围 1~247
6. **恢复出厂设置:** CMD 0x0E 清除所有 NVS 含地址修改和设备使能
7. **固件版本:** CMD 0x08 取自 main.c 的 firmware_rev_val (如 "V1.1.3")
8. **SNTP 状态:** CMD 0x05 D13/D14 返回实时状态; CMD 0x26 用于 App 初始化时读取开关状态; SNTP 需 WiFi 已连接才能同步
