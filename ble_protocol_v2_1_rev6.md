# CTM BLE 通信协议文档

**版本:** V2.1 (Rev.6)  
**适用设备:** CTM Modbus 控制器 (ESP32-C3)  
**通信方式:** BLE 5.0 (Nordic UART Service)  
**更新日期:** 2026-05-11

---

## 1. BLE 服务信息

| 项目 | 内容 |
|------|------|
| 设备广播名 | CTM=XXXXXXXXXXXX (12位蓝牙MAC HEX大写) |
| NUS Service UUID | 6E400001-B5A3-F393-E0A9-E50E24DCCA9E |
| RX 特征 UUID | 6E400002-... (Write/WriteNR, App->设备) |
| TX 特征 UUID | 6E400003-... (Notify, 设备->App) |
| MTU | 建议 247+, 最大 500 |
| 配网方式 | BLE NUS 指令 (CMD 0x0B) / SmartConfig (CMD 0x0C) |

**连接流程:**

1. 扫描广播, 过滤名称前缀 CTM=
2. 连接设备
3. 发现 NUS Service
4. 使能 TX Notify (写 CCC = 0x0001)
5. 通过 RX 发送指令帧, 通过 TX Notify 接收应答帧

设备断开后自动恢复广播, App 可重新扫描连接.

---

## 2. 帧格式

```
HEAD(0x68) + CMD(1B) + LEN(2B小端) + DATA(NB) + CS(1B) + TAIL(0x16)
```

| 字段 | 长度 | 说明 |
|------|------|------|
| HEAD | 1 | 固定 0x68 |
| CMD  | 1 | 指令码; 应答帧 CMD 原样返回或为 0x80 (错误) |
| LEN  | 2 | 数据域字节数, 小端 |
| DATA | N | 数据域 |
| CS   | 1 | SUM(HEAD ~ DATA末字节) & 0xFF |
| TAIL | 1 | 固定 0x16 |

## 4. 指令详情

### 4.1 MQTT 开关 (CMD: 0x01)

请求 DATA (1B): `0x00`=关闭, `0x01`=开启  
应答 DATA (1B): `0x00`=成功, `0x0F`=失败

### 4.2 TG服务器 开关 (CMD: 0x02)

格式与 CMD 0x01 相同.  
TG服务器默认启用. 无密钥时状态码为 `0x07`.

### 4.3 读取 MQTT 配置 (CMD: 0x03)

请求 DATA: 无  
应答 DATA (210B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| 0    | 80   | URI (ASCII, 0填充) |
| 80   | 2    | 端口号 (uint16 小端) |
| 82   | 32   | 用户名 (ASCII, 0填充) |
| 114  | 32   | 密码 (ASCII, 0填充) |
| 146  | 64   | Topic (ASCII, 0填充) |

### 4.4 写入 MQTT 配置 (CMD: 0x04)

请求 DATA (210B): 布局同 CMD 0x03  
应答 DATA (1B): `0x00`=成功, `0x0F`=失败 (长度不足或URI为空)

URI 不可为空; port=0 时固件自动使用 1883.

### 4.5 读取全部连接状态 (CMD: 0x05)

请求 DATA: 无  
应答 DATA (17B):

| 偏移   | 长度 | 说明 |
|--------|------|------|
| D0     | 1    | WiFi 状态 |
| D1     | 1    | WiFi RSSI (int8, dBm) |
| D2     | 1    | MQTT 使能: 0=禁用, 1=启用 |
| D3     | 1    | MQTT 状态 |
| D4     | 1    | TG服务器 使能: 0=禁用, 1=启用 |
| D5     | 1    | TG服务器 连接状态 |
| D6     | 1    | Modbus 通讯状态 |
| D7-D10 | 4    | MQTT 上次上报时间 (uint32 小端, Unix秒) |
| D11    | 1    | TG密钥配置: 0=未配置, 1=已配置 |
| D12    | 1    | TCP综合配置状态: 0=未配置, 1=已配密钥(默认TG), 2=完整配置 |
| D13    | 1    | 每日校时使能: 0=关闭, 1=开启 |
| D14    | 1    | SNTP 同步状态 |
| D15    | 1    | TCP 服务器模式: 0=自定义TCP, 1=TG服务器 |
| D16    | 1    | TCP 加密状态: 0=明文, 1=AES加密 |

**WiFi 状态码:**

| 值 | 含义 |
|----|------|
| 0x00 | 未配置 |
| 0x01 | 连接中 |
| 0x02 | 已连接 |
| 0x03 | 密码错误 |
| 0x04 | AP未找到 |
| 0x05 | 认证失败 |
| 0x06 | 连接超时 |
| 0x07 | SmartConfig 进行中 |
| 0x08 | BLE 配网进行中 |

**MQTT 状态码:**

| 值 | 含义 |
|----|------|
| 0x00 | 已禁用 |
| 0x01 | DNS 失败 |
| 0x02 | TCP 连接失败 |
| 0x03 | 鉴权失败 |
| 0x04 | 连接中 |
| 0x05 | 已连接 |
| 0x06 | 已断开 |

**TG服务器 连接状态码:**

| 值 | 含义 |
|----|------|
| 0x00 | 已禁用 |
| 0x01 | DNS 失败 |
| 0x02 | 连接失败 |
| 0x03 | 连接中 |
| 0x04 | 已连接 |
| 0x05 | 已断开 |
| 0x06 | 等待 WiFi |
| 0x07 | 无密钥 |

**Modbus 通讯状态码:**

| 值 | 含义 |
|----|------|
| 0x00 | 空闲 |
| 0x01 | 通信正常 |
| 0x02 | 超时 |
| 0x03 | CRC 错误 |
| 0x04 | 异常 |

**SNTP 同步状态码:**

| 值 | 含义 |
|----|------|
| 0x00 | 未同步 (SNTP 未启用或从未同步) |
| 0x01 | 已同步 |
| 0x02 | 同步中 |
| 0x03 | 同步失败 (网络不可达) |

### 4.6 读取 MQTT 上报间隔 (CMD: 0x06)

请求 DATA: 无  
应答 DATA (4B): uint32 小端, 单位秒

### 4.7 写入 MQTT 上报间隔 (CMD: 0x07)

请求 DATA (4B): uint32 小端, 范围 10~360 秒  
应答 DATA (1B): `0x00`=成功, `0x0F`=失败

### 4.8 读取设备信息 (CMD: 0x08)

请求 DATA: 无  
应答 DATA (46B):

| 偏移    | 长度 | 说明 |
|---------|------|------|
| D0-D5   | 6    | BLE MAC 地址 (6字节, 大端) |
| D6-D37  | 32   | 固件版本字符串 (ASCII, 如 "V1.1.3", 0填充) |
| D38-D41 | 4    | 运行时间 (uint32 小端, 秒) |
| D42-D45 | 4    | 设备当前时间 (uint32 小端, Unix秒; 0=NTP未同步) |

### 4.9 读取 TCP服务器 配置 (CMD: 0x09)

请求 DATA: 无  
应答 DATA (84B):

| 偏移  | 长度 | 说明 |
|-------|------|------|
| 0-79  | 80   | 服务器主机名/IP (ASCII, 0填充) |
| 80-81 | 2    | 端口号 (uint16 小端) |
| 82    | 1    | server_mode: 0=自定义TCP, 1=TG服务器 |
| 83    | 1    | encrypt_enabled: 0=明文, 1=AES加密 |

### 4.10 写入 TCP服务器 配置 (CMD: 0x0A)

请求 DATA (84B): 布局同 CMD 0x09  
应答 DATA (1B): `0x00`=成功, `0x0F`=失败

写入后持久化, 若服务当前已启用则自动重连.

- **自定义 TCP 模式 (server_mode=0):** 默认 `tgc-tcp-uat.capax-tech.com:9000`, 不需要 app_key, 加密由 encrypt_enabled 控制.
- **TG 模式 (server_mode=1):** 默认 `smartdevice.towngas.com:10020`, 需要 app_key (通过 SmartConfig 写入), 强制 AES 加密 (encrypt_enabled 写入值被忽略, 固件强制置 1).

### 4.11 WiFi 配网 (CMD: 0x0B)

请求 DATA (变长):

| 偏移 | 长度 | 说明 |
|------|------|------|
| 0    | 1    | SSID 长度 N (1~32) |
| 1    | N    | SSID |
| 1+N  | 1    | 密码长度 M (0~64) |
| 2+N  | M    | 密码 (M=0 表示开放网络) |

应答 DATA (1B): `0x00`=已开始, `0x0F`=参数错误

> 注意: BLE 配网不写入 TG 密钥. TG 密钥须通过 SmartConfig (CMD 0x0C) 写入.

### 4.12 触发 SmartConfig (CMD: 0x0C)

请求 DATA: 无  
应答 DATA (1B): `0x00`

SmartConfig (ESPTouch V2) 成功后自动将 rvd_data 写入 app_key, 同时连接 WiFi.

### 4.13 设备重启 (CMD: 0x0D)

请求 DATA: 无  
应答 DATA (1B): `0x00`, 应答发出后 500ms 重启.

### 4.14 恢复出厂设置 (CMD: 0x0E)

请求 DATA: 无  
应答 DATA (1B): `0x00`

清除全部 NVS 数据 (含 WiFi 配置、TG 密钥、Modbus 地址、服务配置等) 并重启. **不可逆, App 应做二次确认.**

### 4.15 读取 TG服务器 上报间隔 (CMD: 0x10)

请求 DATA: 无  
应答 DATA (4B): uint32 小端, 单位秒

### 4.16 写入 TG服务器 上报间隔 (CMD: 0x11)

请求 DATA (4B): uint32 小端, 范围 10~360 秒  
应答 DATA (1B): `0x00`=成功, `0x0F`=失败

### 4.17 读取 Modbus 设备状态 (CMD: 0x20)

请求 DATA: 无  
应答 DATA (145B):

| 偏移 | 长度   | 说明 |
|------|--------|------|
| D0   | 1      | 设备类型数量 N (固定 9) |
| D1~  | N × 16 | 每设备 16 字节 |

**每设备 16 字节结构:**

| 偏移    | 长度 | 说明 |
|---------|------|------|
| +0~+7   | 8    | 设备名称 (ASCII, 0填充) |
| +8      | 1    | 从机地址 |
| +9      | 1    | 使能: 0=禁用, 1=启用 |
| +10     | 1    | 通讯状态: 0=空闲, 1=正常, 2=超时, 3=CRC, 4=异常 |
| +11     | 1    | 保留 |
| +12~+13 | 2    | 当前寄存器起始地址 (uint16 小端) |
| +14~+15 | 2    | 当前寄存器数量 (uint16 小端) |

**设备类型列表:**

| 索引 | 名称      | 默认从机地址 | 默认起始地址 | 默认寄存器数 |
|------|-----------|-------------|-------------|-------------|
| 0    | GAS_FLOW  | 0x01        | 0x0001      | 13          |
| 1    | FRIDGE    | 0x02        | 0x0000      | 21          |
| 2    | CURRENTWA | 0x03        | 0x0020      | 6           |
| 3    | DRYCONTAC | 0x04        | 0x0000      | 3           |
| 4    | DISHWASHE | 0x08        | 0x1320      | 42          |
| 5    | GAS_CONC  | 0x09        | 0x0016      | 10          |
| 6    | ULTRASONI | 0x0A        | 0x0000      | 8           |
| 7    | STEAMER   | 0x19        | 0x0000      | 5           |
| 8    | WOK       | 0x1A        | 0x0000      | 3           |

### 4.18 设置 Modbus 设备通讯使能 (CMD: 0x21)

请求 DATA (2B): D0=设备索引 (0~8), D1=使能 (0=禁用, 1=启用)  
应答 DATA (1B): `0x00`=成功, `0x0F`=失败 (索引越界)

持久化到 NVS, 立即生效.

### 4.19 读取 Modbus 通讯日志 (CMD: 0x22)

请求 DATA: 无  
应答 DATA (变长, 最大 321B):

| 偏移 | 长度   | 说明 |
|------|--------|------|
| D0   | 1      | 日志条数 N (0~10) |
| D1~  | N × 32 | 每条 32 字节 |

**每条日志 32 字节结构:**

| 偏移    | 长度 | 说明 |
|---------|------|------|
| +0~+3   | 4    | 时间戳 (uint32 小端, Unix秒) |
| +4~+19  | 16   | 设备名称 (ASCII, 0填充) |
| +20     | 1    | 通讯结果: 0=成功, 1=超时, 2=CRC错误, 3=异常响应 |
| +21     | 1    | 从机地址 |
| +22     | 1    | 操作类型: 0=读保持寄存器, 1=读输入寄存器, 2=读离散输入, 3=写保持寄存器 |
| +23     | 1    | 保留 |
| +24~+25 | 2    | 寄存器起始地址 (uint16 小端) |
| +26~+27 | 2    | 寄存器数量 (uint16 小端) |
| +28~+31 | 4    | 保留 |

App 显示建议: 用图标区分操作类型; 结果绿色=成功, 红色=失败; 寄存器信息格式 `Addr: 0x0001, Cnt: 13`.

### 4.20 修改 Modbus 寄存器地址 (CMD: 0x23)

请求 DATA (5B):

| 偏移  | 长度 | 说明 |
|-------|------|------|
| D0    | 1    | 设备索引 (0~8) |
| D1-D2 | 2    | 新起始地址 (uint16 小端) |
| D3-D4 | 2    | 新寄存器数量 (uint16 小端, 1~125) |

应答 DATA (1B): `0x00`=成功, `0x02`=数量超范围, `0x0F`=索引越界

持久化到 NVS, 下一个轮询周期立即生效.

### 4.21 修改 Modbus 从机地址 (CMD: 0x24)

请求 DATA (2B): D0=设备索引 (0~8), D1=新从机地址 (1~247)  
应答 DATA (1B): `0x00`=成功, `0x02`=地址超范围, `0x0F`=索引越界

持久化到 NVS, 下一个轮询周期立即生效.

### 4.22 每日校时开关 (CMD: 0x25)

请求 DATA (1B): `0x00`=关闭, `0x01`=开启  
应答 DATA (1B): `0x00`=成功, `0x0F`=失败

控制设备是否启用每日 SNTP 自动校时. 持久化到 NVS.

### 4.23 读取校时状态 (CMD: 0x26)

请求 DATA: 无  
应答 DATA (2B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0   | 1    | 每日校时使能: 0=关闭, 1=开启 |
| D1   | 1    | SNTP 同步状态 (同 CMD 0x05 D14) |

App 进入设置页面时调用此指令初始化开关 UI 状态.

### 4.24 读取当前 WiFi 配置 (CMD: 0x27)

请求 DATA: 无  
应答 DATA (97B):

| 偏移    | 长度 | 说明 |
|---------|------|------|
| D0      | 1    | SSID 长度 N (0~32; 0=未配置) |
| D1-D32  | 32   | SSID (UTF-8, 0填充) |
| D33     | 1    | 密码长度 M (0~64) |
| D34-D97 | 64   | 密码 (UTF-8, 0填充) |

### 4.25 读取 Modbus 轮询间隔 (CMD: 0x28)

请求 DATA: 无  
应答 DATA (4B): uint32 小端, 单位毫秒 (默认 2000)

### 4.26 写入 Modbus 轮询间隔 (CMD: 0x29)

请求 DATA (4B): uint32 小端, 单位毫秒, 范围 500~10000  
应答 DATA (1B): `0x00`=成功, `0x02`=超范围, `0x0F`=失败

持久化到 NVS, 立即生效.

### 4.27 读取最近 MQTT 上报报文 (CMD: 0x2A)

读取设备最近缓存的 MQTT 上报 JSON 报文, 最多 5 条, 按时间倒序 (最新在前).

请求 DATA: 无  
应答 DATA (变长, 最大 1311B):

| 偏移 | 长度    | 说明 |
|------|---------|------|
| D0   | 1       | 报文条数 N (0~5) |
| D1~  | N × 变长 | 每条报文 |

**每条报文结构:**

| 偏移        | 长度 | 说明 |
|-------------|------|------|
| +0~+3       | 4    | 发送时间戳 (uint32 小端, Unix秒; 0=设备时间未同步) |
| +4~+5       | 2    | JSON 长度 L (uint16 小端, 最大 256) |
| +6~+(L+5)   | L    | JSON 报文 (UTF-8, 不含 `\0`) |

**当前 JSON 格式示例:**
```json
{"Device ID":"AABBCCDDEEFF","Instantflow":"7.809","Totalflow":"3975.115"}
```

App 显示建议: 以列表展示, 每条显示格式化时间 (`YYYY-MM-DD HH:MM:SS`) 和 JSON 内容, 等宽字体, 支持长按复制.

### 4.28 OTA 开始 (CMD: 0x70)

请求 DATA (4B): 固件大小 (uint32 **大端**)  
应答 DATA (1B): `0x00`=成功, `0x01`=分区错误, `0x02`=固件超出分区大小

### 4.29 OTA 数据 (CMD: 0x71)

请求 DATA (N B): 固件二进制分块数据 (每包建议 ≤ MTU - 10)  
应答 DATA (5B):

| 偏移  | 长度 | 说明 |
|-------|------|------|
| +0    | 1    | 结果: `0x00`=成功, `0x02`=写入失败, `0xFF`=无活动会话 |
| +1~+4 | 4    | 已写入字节数 (uint32 **大端**) |

### 4.30 OTA 执行 (CMD: 0x72)

请求 DATA: 无  
应答 DATA (1B): `0x00`=成功并重启, `0x01`=校验失败, `0x03`=数据不完整, `0xFF`=无活动会话

## 5. WiFi 配网流程

### 5.1 BLE 配网 (CMD 0x0B)

1. App 发送 CMD 0x0B, 携带 SSID 和密码
2. 设备断开当前 WiFi, 以新配置重新连接
3. App 轮询 CMD 0x05 D0, 等待 WiFi 状态变为 `0x02` (已连接)
4. 连接成功后 TCP/MQTT 服务自动启动

> BLE 配网**不写入 TG 密钥**, TG 服务器模式须先通过 SmartConfig 配网.

### 5.2 SmartConfig 配网 (CMD 0x0C)

1. App 发送 CMD 0x0C, 设备进入 SmartConfig 模式
2. App 使用 ESPTouch V2 协议广播 SSID + 密码 + TG 密钥 (rvd_data)
3. 设备收到后自动写入 app_key, 连接 WiFi
4. 连接成功后 TCP/MQTT 服务自动启动

## 6. OTA 升级流程

1. App 发送 CMD 0x70, 携带固件大小
2. 循环发送 CMD 0x71, 每包建议 ≤ (MTU - 10)B, 根据应答中的已写入字节数判断进度
3. 全部数据发送完毕后发送 CMD 0x72
4. 设备校验通过后应答 `0x00` 并在 500ms 内重启

---

## 7. TG 密钥机制

- app_key 是 32 字节 AES 密钥, 通过 SmartConfig (ESPTouch V2) 的 rvd_data 写入设备 NVS
- TG 模式下 TCP 连接建立后, 设备先发送密钥请求帧 (type=0xFF), 服务器下发加密后的 Remote Key 和 IV
- 密钥交换完成后才进入正常上报状态
- 无 app_key 时 TG 服务器状态码为 `0x07` (无密钥), 无法连接
- BLE 配网 (CMD 0x0B) 不写入 app_key, 必须使用 SmartConfig

---

## 8. Modbus 设备管理

- **CMD 0x20** 一次性读取所有 9 个设备的名称、从机地址、使能状态、通讯状态和当前寄存器配置
- **CMD 0x21** 控制各设备是否参与轮询, 持久化
- **CMD 0x22** 查看最近 10 条通讯事件, 含操作类型、寄存器地址/数量和结果
- **CMD 0x23** 修改寄存器起始地址和数量, 持久化 + 立即生效; 数量范围 1~125
- **CMD 0x24** 修改从机地址, 持久化 + 立即生效; 地址范围 1~247
- **CMD 0x28/0x29** 读取/写入 Modbus 轮询间隔, 范围 500~10000ms

## 9. 通用错误码

| 值   | 含义 |
|------|------|
| 0x00 | 成功 |
| 0x01 | 分区/校验错误 |
| 0x02 | 空间不足 / 参数无效 |
| 0x03 | 数据不完整 |
| 0x0F | 参数错误 / 数据长度不足 / 索引越界 |
| 0xFF | 未知错误 / 无活动 OTA 会话 |

## 10. 注意事项

1. **字节序:** 所有多字节整数均为小端; OTA 固件大小和已写入字节数为大端
2. **状态轮询:** CMD 0x05 建议间隔 2~3 秒
3. **上报间隔:** MQTT (CMD 0x06/0x07) 和 TG服务器 (CMD 0x10/0x11) 各自独立, 均为 10~360 秒
4. **TG 密钥:** 必须通过 SmartConfig 写入, BLE 配网不写入密钥
5. **Modbus 修改:** CMD 0x23/0x24 均持久化到 NVS 并立即生效; 恢复出厂设置 (CMD 0x0E) 后恢复默认值
6. **设备时间:** CMD 0x08 D42-D45 和 CMD 0x2A 时间戳均依赖 SNTP 同步, 未同步时为 0
7. **固件版本:** CMD 0x08 固件版本取自 `main.c` 的 `firmware_rev_val`
