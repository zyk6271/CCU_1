# CTM BLE 通信协议文档

**版本:** V2.1 (Rev.2)  
**适用设备:** CTM Modbus 控制器 (ESP32-C3)  
**通信方式:** BLE 5.0 (Nordic UART Service)  
**更新日期:** 2026-04-28

---

## 1. BLE 服务信息

| 项目 | 内容 |
|------|------|
| 设备广播名 | CTM=XXXXXXXXXXXX (12位蓝牙MAC HEX大写) |
| NUS Service UUID | 6E400001-B5A3-F393-E0A9-E50E24DCCA9E |
| RX 特征 UUID | 6E400002-... (Write/WriteNR, App->设备) |
| TX 特征 UUID | 6E400003-... (Notify, 设备->App) |
| MTU | 建议 247+, 最大 500 |
| 配网方式 | BLE NUS 指令 (CMD 0x0B) / SmartConfig (CMD 0x0C) |

**连接流程:**

1. 扫描广播, 过滤名称前缀 CTM=
2. 连接设备
3. 发现 NUS Service
4. 使能 TX Notify (写 CCC = 0x0001)
5. 通过 RX 发送指令帧, 通过 TX Notify 接收应答帧

设备断开后自动恢复广播, App 可重新扫描连接.

---

## 2. 帧格式

```
HEAD(0x68) + CMD(1B) + LEN(2B小端) + DATA(NB) + CS(1B) + TAIL(0x16)
```

| 字段 | 长度 | 说明 |
|------|------|------|
| HEAD | 1 | 固定 0x68 |
| CMD | 1 | 指令码, 应答帧 CMD 或 0x80 |
| LEN | 2 | 数据域长度, 小端 |
| DATA | N | 数据域 |
| CS | 1 | SUM(HEAD ~ DATA末字节) & 0xFF |
| TAIL | 1 | 固定 0x16 |

## 4. 指令详情

### 4.1 MQTT 开关 (CMD: 0x01)

请求 DATA (1B): D0: 0x00=关闭, 0x01=开启

应答 DATA (1B): D0: 0x00=成功, 0x0F=失败

设置持久化到 NVS.

### 4.2 TCP 开关 (CMD: 0x02)

格式与 CMD 0x01 完全相同.

注意: TCP 服务默认启用. 如果设备未写入过 TCP 密钥(app_key), 即使已启用, 状态码会显示 0x07(无密钥). 密钥通过 SmartConfig 配网时自动写入.

### 4.3 读取 MQTT 配置 (CMD: 0x03)

请求 DATA: 无

应答 DATA (210B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0-D79 | 80 | URI (0填充, 如 mqtt://192.168.1.1) |
| D80-D81 | 2 | 端口, 小端 (如 1883 = 0x5B 0x07) |
| D82-D113 | 32 | 用户名 (0填充) |
| D114-D145 | 32 | 密码 (0填充) |
| D146-D209 | 64 | Topic (0填充) |

### 4.4 写入 MQTT 配置 (CMD: 0x04)

请求 DATA (210B): 布局同 CMD 0x03 应答.

应答 DATA (1B): 0x00=成功, 0x0F=长度不足

写入后持久化到 NVS. MQTT 运行中会自动重启生效.

### 4.5 读取全部连接状态 (CMD: 0x05) ★ Rev.2

App 定时轮询此指令获取全部状态, 建议间隔 2~3 秒.

请求 DATA: 无

应答 DATA (**11B**):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | WiFi 状态 |
| D1 | 1 | WiFi RSSI (int8, dBm, 未连接时为 0) |
| D2 | 1 | MQTT 使能: 0=禁用, 1=启用 |
| D3 | 1 | MQTT 状态 |
| D4 | 1 | TCP 使能: 0=禁用, 1=启用 |
| D5 | 1 | TCP 状态 |
| **D6** | **1** | **Modbus 连接状态** |
| **D7-D10** | **4** | **MQTT 上次上报时间 (uint32 小端, Unix时间戳秒, 0=从未上报)** |

**WiFi 状态码:**

| 值 | 状态 | App 显示建议 |
|----|------|-------------|
| 0x00 | 未配置 | "未配置WiFi" |
| 0x01 | 连接中 | "WiFi连接中..." |
| 0x02 | 已连接 | "WiFi已连接" + 显示RSSI |
| 0x03 | 密码错误 | "WiFi密码错误, 请重新输入" |
| 0x04 | AP未找到 | "未找到该WiFi网络" |
| 0x05 | 认证失败 | "WiFi认证失败" |
| 0x06 | 连接超时 | "WiFi连接超时" |
| 0x07 | SmartConfig中 | "SmartConfig配网中..." |
| 0x08 | BLE配网中 | "WiFi连接中..." |

**MQTT 状态码:**

| 值 | 状态 | App 显示建议 |
|----|------|-------------|
| 0x00 | 已禁用 | "MQTT未启用" |
| 0x01 | DNS失败 | "MQTT服务器域名解析失败" |
| 0x02 | TCP连接失败 | "MQTT服务器连接失败(端口/防火墙)" |
| 0x03 | 鉴权失败 | "MQTT用户名或密码错误" |
| 0x04 | 连接中 | "MQTT连接中..." |
| 0x05 | 已连接 | "MQTT已连接" |
| 0x06 | 已断开 | "MQTT连接已断开" |

**TCP 状态码:**

| 值 | 状态 | App 显示建议 |
|----|------|-------------|
| 0x00 | 已禁用 | "TCP未启用" |
| 0x01 | DNS失败 | "TCP服务器域名解析失败" |
| 0x02 | 连接失败 | "TCP服务器连接失败" |
| 0x03 | 连接中 | "TCP连接中..." |
| 0x04 | 已连接 | "TCP已连接" |
| 0x05 | 已断开 | "TCP连接已断开" |
| 0x06 | 等待WiFi | "等待WiFi连接..." |
| 0x07 | 无密钥 | "未配网, 请先通过SmartConfig写入密钥" |

**Modbus 连接状态码:**

| 值 | 状态 | App 显示建议 |
|----|------|-------------|
| 0x00 | 空闲/未启用 | "空闲/未启用" |
| 0x01 | 通信正常 | "通信正常" |
| 0x02 | 通信超时 | "通信超时(从机无响应)" |
| 0x03 | CRC校验错误 | "CRC校验错误" |
| 0x04 | 从机异常响应 | "从机异常响应" |

### 4.6 读取 MQTT 上报间隔 (CMD: 0x06)

请求 DATA: 无

应答 DATA (4B): uint32 小端, 单位秒

### 4.7 写入 MQTT 上报间隔 (CMD: 0x07) ★ Rev.2 范围变更

请求 DATA (4B): uint32 小端, 范围 **10~360**

应答 DATA (1B): 0x00=成功, 0x0F=失败

### 4.8 读取设备信息 (CMD: 0x08)

请求 DATA: 无

应答 DATA (38B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0-D5 | 6 | BLE MAC 地址 (6字节原始值) |
| D6-D37 | 32 | 固件版本 (ASCII, 如 "1.2.0", 0填充) |

App 拿到 MAC 后按 %02X:%02X:%02X:%02X:%02X:%02X 格式显示.

### 4.9 读取 TCP 配置 (CMD: 0x09)

请求 DATA: 无

应答 DATA (84B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0-D79 | 80 | 服务器地址/域名 (0填充) |
| D80-D81 | 2 | 端口, 小端 |
| D82-D83 | 2 | 保留 |

### 4.10 写入 TCP 配置 (CMD: 0x0A)

请求 DATA (82~84B): Host(80) + Port(2, 小端) + Reserved(2, 可选)

应答 DATA (1B): 0x00=成功, 0x0F=长度不足

写入后持久化, TCP 运行中自动断开并以新配置重连.

### 4.11 WiFi 配网 (CMD: 0x0B)

请求 DATA (变长):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | SSID 长度 N (1~32) |
| D1 ~ D[N] | N | SSID (UTF-8, 不含 \0) |
| D[N+1] | 1 | 密码长度 M (0~64, 0=开放网络) |
| D[N+2] ~ D[N+1+M] | M | 密码 (不含 \0) |

应答 DATA (1B): 0x00=已收到开始连接, 0x0F=参数错误

配网结果通过轮询 CMD 0x05 查看 WiFi 状态:
- D0=0x02 表示配网成功
- D0=0x03 表示密码错误
- D0=0x04 表示AP未找到
- D0=0x08 表示还在连接中, 继续等待

注意: BLE 配网不会写入 TCP 密钥(app_key). 如果需要连接 towngas TCP 服务器, 必须使用 SmartConfig 配网.

示例 (SSID=MyWiFi, Password=12345678):
```
发送: 68 0B 10 00 06 4D 79 57 69 46 69 08 31 32 33 34 35 36 37 38 CS 16
                  |SSID长度=6|--- "MyWiFi" --|密码长度=8|--- "12345678" --|
应答: 68 8B 01 00 00 CS 16    (已开始连接)
```

### 4.12 触发 SmartConfig (CMD: 0x0C)

请求 DATA: 无

应答 DATA (1B): 0x00=已启动

启动后 CMD 0x05 的 WiFi 状态变为 0x07 (SmartConfig中). 3 分钟超时自动停止.

SmartConfig 配网成功后会自动写入 TCP 密钥(app_key), 此后 TCP 服务即可正常连接.

### 4.13 设备重启 (CMD: 0x0D)

请求 DATA: 无

应答 DATA (1B): 0x00

应答后 500ms 设备重启, BLE 连接断开, App 需重新扫描连接.

### 4.14 恢复出厂设置 (CMD: 0x0E)

请求 DATA: 无

应答 DATA (1B): 0x00

应答后设备清除全部 NVS 数据 (WiFi 凭证、MQTT/TCP 配置、服务开关、上报间隔、TCP 密钥) 并重启.

**不可逆操作, App 应弹出二次确认对话框.**

### 4.15 读取 TCP 上报间隔 (CMD: 0x10)

请求 DATA: 无

应答 DATA (4B): uint32 小端, 单位秒

### 4.16 写入 TCP 上报间隔 (CMD: 0x11) ★ Rev.2 范围变更

请求 DATA (4B): uint32 小端, 范围 **10~360**

应答 DATA (1B): 0x00=成功, 0x0F=失败

### 4.17 OTA 开始 (CMD: 0x70)

请求 DATA (4B): 固件文件大小 (字节), **大端**

应答 DATA (1B): 0x00=就绪, 0x01=分区错误, 0x02=空间不足, 0xFF=参数错误

### 4.18 OTA 数据 (CMD: 0x71)

请求 DATA: 纯固件数据块 (建议每包 240B 以内)

应答 DATA (5B):

| 偏移 | 长度 | 说明 |
|------|------|------|
| D0 | 1 | 0x00=成功, 0x02=写入失败, 0xFF=无会话 |
| D1-D4 | 4 | 已接收总字节数 (大端) |

### 4.19 OTA 执行 (CMD: 0x72)

请求 DATA: 无

应答 DATA (1B): 0x00=成功(500ms后重启), 0x01=校验失败, 0x02=分区失败, 0x03=数据不完整

## 5. WiFi 配网流程 (App 端)

```
1. BLE 连接设备, 使能 Notify
2. 发送 CMD 0x0B (SSID + Password)
3. 收到应答 0x00 -> 弹出进度弹窗
4. 每 2 秒轮询 CMD 0x05
5. 判断 D0 (WiFi状态):
   - 0x02 (已连接) -> 配网成功
   - 0x03 (密码错误) -> 提示用户重新输入
   - 0x04 (AP未找到) -> 提示检查WiFi名称
   - 0x08 (BLE配网中) -> 继续等待
6. 超时 30 秒无结果 -> 提示连接超时
```

---

## 6. OTA 升级流程

```
1. 发送 CMD 0x70 (固件大小, 大端) -> 应答 0x00 就绪
2. 循环发送 CMD 0x71 (每包 <= 240B) -> 应答中确认进度
3. 全部发完, 发送 CMD 0x72 -> 应答 0x00
4. 设备 500ms 后重启, BLE 断开
```

OTA 前务必先协商 MTU 到 247+, 否则每包仅 14 字节.

## 7. TCP 密钥机制说明

设备连接 towngas TCP 服务器需要加密通信, 密钥 (app_key, 32字节) 的来源和规则:

- SmartConfig (ESPTouch V2) 配网成功时, 手机 App 通过 reserved data 字段下发密钥, 设备自动存入 NVS
- BLE 配网 (CMD 0x0B) 不涉及密钥传输, 仅写入 WiFi 凭证
- 设备启动时检查 NVS 中是否存在有效的 app_key (非全零, 长度32字节)
- 无密钥时: TCP 服务无法启动, 状态码固定为 0x07 (无密钥)
- 恢复出厂设置 (CMD 0x0E) 会清除密钥, 之后需重新 SmartConfig

App 端建议: 如果 TCP 状态持续显示 0x07, 提示用户 "请使用 SmartConfig 配网以启用 TCP 服务".

## 8. 通用错误码

| 值 | 含义 |
|----|------|
| 0x00 | 成功 |
| 0x01 | 分区/校验错误 |
| 0x02 | 空间不足/连接失败 |
| 0x03 | 数据不完整 |
| 0x0F | 参数错误/数据长度不足 |
| 0xFF | 未知错误/无活动OTA会话 |

## 9. 变更记录

| 版本 | 变更项 | 说明 |
|------|--------|------|
| Rev.1 | CMD 0x05 应答 | 6B → 7B, 新增 D6 Modbus 连接状态 |
| Rev.2 | CMD 0x05 应答 | 7B → **11B**, 新增 D7-D10 MQTT 上次上报时间戳 |
| Rev.2 | CMD 0x07/0x11 | 上报间隔范围 10~86400 → **10~360** |

App 端兼容: 解析 CMD 0x05 时按 `data.length` 判断可用字段:
- `>= 6`: WiFi + MQTT + TCP 状态 (基础)
- `>= 7`: + Modbus 状态
- `>= 11`: + MQTT 上次上报时间

## 10. 注意事项

1. **字节序:** 所有多字节整数小端, OTA 文件大小和进度为大端
2. **状态轮询:** CMD 0x05 返回 WiFi + MQTT + TCP + Modbus + MQTT上报时间, App 定时 2~3 秒轮询
3. **上报间隔:** MQTT (CMD 0x06/0x07) 和 TCP (CMD 0x10/0x11) 各自独立, 范围 10~360 秒
4. **TCP 默认启用:** TCP 服务始终启用, 无需 App 开关控制
5. **TCP 密钥:** 必须通过 SmartConfig 写入, BLE 配网不会写入密钥, 无密钥时 TCP 不可用
6. **配网共存:** BLE 配网 (0x0B) 会自动停止 SmartConfig; SmartConfig (0x0C) 3 分钟超时
7. **恢复出厂:** CMD 0x0E 清除所有 NVS (含 TCP 密钥), 不可逆, App 务必做二次确认
8. **设备重启/OTA:** CMD 0x0D / 0x72 应答后 500ms 断连, App 需重新扫描
9. **MQTT 状态细分:** DNS失败/TCP连接失败/鉴权失败 各有独立状态码, 便于 App 引导用户修正配置
10. **持久化:** WiFi 凭证、MQTT 配置、TCP 配置、服务开关、上报间隔均存 NVS, 掉电保持
